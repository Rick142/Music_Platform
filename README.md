BACKEND
ТЕОРИЯ NEST JS
Nest JS - обёртка на фреймворк Express для Node JS. Он состоит из контроллера, модели и провайдера (service). Названия всех трёх строятся особым образом, например:

app.controller.ts
app.module.ts
app.service.ts
app.schema.ts

КОНТРОЛЛЕР
Контроллер - файл, в котором мы взаимодействуем с запросами и ответами. Запросы прописываются с помощью декораторов, например @Get. Внутри декораторов создаются функции, возвращающие функциии из сервиса, которые помещаются в контроллер с помощью определения зависимостей (конструктора). Возврат функций внутри декораторов проходит с помощью this.file_name.func(). Также внутри контроллеров работаем с входными параметрами (query, body, headers). Те вздесь вся логика с запросами и ответами, клиент-серверная составляющая.

МОДУЛЬ
Модуль - файл, в котором мы регистрируем контроллеры и провайдеры. Регистрация проходит с помощью нотаций. Сам модуль мы передаём в базовую конфигурацию сервера (main.ts), а именно в аргумент функции create(). Также в модуле можно регистрировать модули, стоящие ниже по иерархии.

ПРОВАЙДЕР (SERVICE)
Провайдер (сервис) - файл, в котором мы описываем какую-либо логику, например получение данных из БД, возврат. Данную логику мы применяем в контроллере, предварительно зарегестрировав сервис в модуле. Для использования сервиса его нужно пометить нотацией @Injectable. Это делается для определения зависимостей. При внедрении сервиса в контроллер создаётся подобный конструктор, где appService - передаваемый нами сервис.

constructor(private appService: AppService){}
